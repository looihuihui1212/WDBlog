What I learnt? Week 6

Javascript

Today, I learned about two types of objects in JavaScript: Literal Objects and Function-Based Objects. Literal Objects are straightforward and have no constructors. They are suitable for creating single instances, like global configurations such as 'game settings.' On the other hand, Function-Based Objects have constructors and are useful for creating multiple instances with similar functionality.

What I learnt? Week 7

DOM

Today, I learned that when a webpage loads on a browser, it creates something called a DOM Tree. This tree is like a model that shows the structure of the webpage, including all the elements like text, images, and buttons. The DOM Tree helps browsers understand and change the content of a webpage, making it possible for websites to be interactive and dynamic. It was interesting to discover how this DOM Tree plays a crucial role in web development, making websites more user-friendly and responsive.

What I learnt? Week 12

RESTDB & NoSQL

Today, I learned that NoSQL databases, also known as "Not only SQL." These databases are great for storing and getting unstructured data. Unlike traditional databases, NoSQL can manage a massive amount of data with a flexible structure called a dynamic schema. In a NoSQL database, there's no fixed way to ask for data (query language), and it has fewer relationships between data. Instead, it stores information in collections and documents. Imagine a library where each book is a document, and all books together make a collection. NoSQL databases allow having many collections, each containing lots of documents, making it versatile for handling different kinds of data.

What I learnt? Week 14

Lottie

Today, I learned about Lottie, a cool tool for adding animations and interactivity to websites or mobile apps. Lottie uses a JSON-based file format for animations, making it easy for designers to share animations on various platforms just like static images. These animation files are compact, work on any device, and can be resized without losing quality. It's like sending small and versatile animated pictures that look great no matter the device or screen size. Extremely Lightweight Animation & Design, because it is just code
